Cohere provides enterprise-grade language model APIs for text generation, embeddings, reranking, and classification. For Game Warden tenants building AI-powered features—such as semantic search, document classification, or conversational interfaces—Cohere offers a straightforward API integration.

## What Is Cohere?

Cohere is an AI company specializing in language models for enterprise use cases. Its API-based platform offers:

- **Command**: Large language models for text generation, summarization, chat, and instruction-following
- **Embed**: High-quality text embeddings for semantic search and retrieval-augmented generation (RAG)
- **Rerank**: Re-ranking of search results for improved relevance
- **Classify**: Zero-shot and few-shot text classification

Cohere models are particularly well-suited for enterprise information retrieval, document processing, and support automation use cases.

## Supported Use Cases on Game Warden

| Use Case | Recommended Model | Notes |
|---|---|---|
| Semantic document search | Embed v3 + Rerank | Use Embed to vectorize documents, Rerank to improve retrieval relevance |
| Chat / conversational AI | Command R+ | Optimized for RAG workflows and multi-turn conversations |
| Document summarization | Command | Efficient summarization of long documents |
| Entity extraction | Command with structured output | Extract structured data from unstructured text |
| Text classification | Classify | Zero-shot classification without fine-tuning |

## Prerequisites

Before integrating Cohere with your GW-deployed application:

- A Cohere account with a valid API key (sign up at [cohere.com](https://cohere.com))
- For production use, a Cohere enterprise license (required for use in federal environments; contact Cohere sales)
- Egress connectivity from your GW namespace to `api.cohere.com` (requires a support ticket to allowlist)
- Game Warden Vault secrets path provisioned for storing your Cohere API key

## Configuring API Credentials Securely

Never hardcode your Cohere API key in application code, environment variable manifests, or container images. Use Game Warden's Vault secrets manager:

<Steps>
  <Step n={1} title="Request a Vault Secret Path">
    Submit a support ticket requesting a Vault secret path for your namespace (e.g., `secret/your-app/cohere`).
  </Step>
  <Step n={2} title="Store Your API Key in Vault">
    Once the path is provisioned, store your Cohere API key:
    ```bash
    vault kv put secret/your-app/cohere api_key="your-cohere-api-key"
    ```
    Second Front will provide Vault CLI access or an alternative method for initial secret population.
  </Step>
  <Step n={3} title="Configure Vault Agent Injection">
    Add Vault Agent annotations to your Kubernetes deployment to inject the API key as an environment variable at pod startup:
    ```yaml
    annotations:
      vault.hashicorp.com/agent-inject: "true"
      vault.hashicorp.com/agent-inject-secret-cohere: "secret/your-app/cohere"
      vault.hashicorp.com/agent-inject-template-cohere: |
        {{ with secret "secret/your-app/cohere" -}}
        export COHERE_API_KEY="{{ .Data.data.api_key }}"
        {{- end }}
    ```
  </Step>
  <Step n={4} title="Access the Key in Your Application">
    Your application reads the API key from the environment variable:
    ```python
    import cohere
    import os

    co = cohere.Client(api_key=os.environ["COHERE_API_KEY"])
    ```
  </Step>
</Steps>

## Example Deployment Pattern: Semantic Search

A common Cohere integration pattern for Game Warden applications is semantic search over a document corpus:

1. **Indexing**: Use `co.embed()` to generate embeddings for each document chunk. Store embeddings in a vector database (e.g., pgvector extension on your GW managed PostgreSQL instance).
2. **Retrieval**: For each user query, generate an embedding and retrieve the top-k most similar document chunks from the vector store.
3. **Reranking**: Pass the top-k results through `co.rerank()` to improve relevance ordering.
4. **Generation**: Optionally pass the reranked results and the user query to `co.chat()` to generate a grounded answer.

## Data Handling Considerations

<Callout type="important">
  Data sent to the Cohere API leaves your Game Warden environment and is transmitted to Cohere's infrastructure. Do not send data classified above what your Cohere agreement and your ATO authorize for external processing. For IL4/IL5 workloads, confirm with your program's Authorizing Official before sending CUI through the Cohere API.
</Callout>

- Use Cohere's enterprise data processing agreements (DPA) to ensure appropriate data handling terms are in place
- Configure your Cohere account to opt out of data training if required by your program
- Implement input filtering to prevent accidental transmission of sensitive data in prompts
- Log all API calls (endpoint, model, token counts) for audit purposes—do not log full prompt/completion text if it may contain sensitive content
