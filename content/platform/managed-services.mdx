Game Warden offers a set of managed platform services that your application can use instead of running and maintaining these components yourself. Using managed services reduces your operational burden, improves your compliance posture (GW's security documentation covers managed services), and provides built-in high availability and backup.

## Available Managed Services

| Service | Type | Availability | Notes |
|---|---|---|---|
| **PostgreSQL** | Relational database | DoW, FedRAMP, Commercial | Version 14 and 15 available. Automated backups, point-in-time recovery. |
| **Redis** | In-memory cache / message broker | DoW, FedRAMP, Commercial | Version 7.x. Persistence optional. Cluster mode available for high-throughput use cases. |
| **Object Storage (S3-compatible)** | Blob/object storage | DoW, FedRAMP, Commercial | S3-compatible API (AWS S3 on Commercial/FedRAMP, GovCloud S3 on DoW). Bucket-level access control. |
| **Secrets Management** | Vault-based secrets | All environments | Dynamic credentials, certificate issuance, key-value secrets. Integrated with Kubernetes via Vault Agent. |
| **Service Mesh** | Istio | All environments (required on IL4/IL5) | mTLS, traffic management, observability. Automatically provisioned per namespace. |
| **Certificate Management** | cert-manager | All environments | Automatic TLS certificate provisioning via Let's Encrypt (Commercial/FedRAMP) or internal PKI (DoW). |

## How to Request a Managed Service

Managed services are requested through App Central or through a support ticket.

<Steps>
  <Step step="1" title="Navigate to Your Application in App Central">
    Open the application that needs the managed service.
  </Step>
  <Step step="2" title="Open the Services Tab">
    Click the **Services** tab on your application detail page.
  </Step>
  <Step step="3" title="Request a New Service">
    Click **Request Service**, select the service type, and fill in the configuration form (database name, storage size, Redis mode, etc.).
  </Step>
  <Step step="4" title="Review and Submit">
    Review the configuration and click **Submit Request**. The Second Front platform team will provision the service, typically within 1 business day.
  </Step>
  <Step step="5" title="Access Credentials">
    Once provisioned, connection credentials are automatically injected into your namespace via Vault. You will receive a notification with the environment variable names to use in your application.
  </Step>
</Steps>

## Limitations and Quotas

Default quotas per application:

| Service | Default Quota |
|---|---|
| PostgreSQL instances | 2 (1 primary, 1 replica) |
| PostgreSQL storage | 100 GB |
| Redis instances | 1 |
| Redis memory | 4 GB |
| Object storage buckets | 5 |
| Object storage capacity | 1 TB |
| Vault secrets paths | 100 |

If your application requires more than the default quotas, submit a support ticket with a justification. Quota increases are reviewed and approved on a case-by-case basis.

<Callout type="note">
  Managed service instances are dedicated to your namespace. They are not shared across tenants. Each instance is provisioned with the security controls and compliance documentation required for your deployment path.
</Callout>
