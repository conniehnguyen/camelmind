These general requirements apply to all applications onboarding to Game Warden, regardless of deployment path (DoW, FedRAMP, or Commercial). They address organizational readiness, operational commitments, and application scope definition.

## Requirements

| Requirement | Description | Notes |
|---|---|---|
| **Containerized application** | Your application must be packaged as one or more OCI-compliant container images. Non-containerized applications (e.g., bare-metal installers, VM images) are not supported. | All services, including background workers and scheduled jobs, must be containerized. |
| **Defined API surface** | Your application must have a documented API surface, including all endpoints, authentication methods, and data inputs/outputs. | OpenAPI/Swagger spec preferred. Internal APIs must also be documented. |
| **Documented dependencies** | All external dependencies (databases, message queues, third-party APIs, etc.) must be documented. | Include version requirements and licensing for all open-source dependencies. |
| **Designated point of contact** | A named technical point of contact (POC) must be designated for onboarding and ongoing communications. | The POC must be reachable via email and phone within 1 business day. |
| **Support SLA** | You must define a support SLA for your application, including response times for critical, high, and medium issues. | The SLA must be at least as stringent as Game Warden's platform SLA for the relevant environment. |
| **Defined system boundary** | You must document the system boundary—what is inside the authorization boundary and what is outside. | Required for BoE completion; start this early. |
| **Unique application identifier** | Your application must have a unique name that does not conflict with existing applications in Game Warden. | App names are validated at submission time. |

## Organizational Requirements

In addition to the application-specific requirements above, your organization must:

- Have a verified account in Game Warden (see [Create Your Account](/getting-started/create-account))
- Have a signed agreement with Second Front Systems
- Have at least one Org Admin designated in the portal

## Checklist Before Submitting

Before submitting your application for entrance criteria review, confirm:

- [ ] All application components are containerized
- [ ] Container images build cleanly and run without errors
- [ ] External dependencies are documented and accessible from the GW environment (or requests for managed services have been submitted)
- [ ] A technical POC is designated and available
- [ ] A support SLA document is ready to attach to your submission
- [ ] Your system boundary is documented (even a rough draft is acceptable at this stage)
