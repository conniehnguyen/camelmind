DoD Impact Levels (ILs) define the sensitivity of data that a cloud system is authorized to handle. The higher the impact level, the stricter the security controls, vetting requirements, and infrastructure constraints. Game Warden supports IL2, IL4, and IL5 deployments, giving vendors a path regardless of their data classification requirements.

## What Are Impact Levels?

Impact Levels are defined by the DoD Cloud Computing Security Requirements Guide (CC SRG). They specify the types of information a cloud environment may process, store, or transmit, and the corresponding security controls that must be in place. Impact Levels are not about the security classification of the data itself (e.g., Secret vs. Unclassified)—they are about the potential impact if that data were compromised.

## Impact Level Comparison

| Impact Level | Data Sensitivity | Who Uses It | Game Warden Support |
|---|---|---|---|
| **IL2** | Publicly releasable, non-sensitive federal data | Federal agencies with no CUI requirement | Supported via FedRAMP path |
| **IL4** | Controlled Unclassified Information (CUI), Covered Defense Information (CDI) | DoD program offices, defense contractors | Fully supported on DoW path |
| **IL5** | National Security Systems (NSS) data, higher-sensitivity CUI | DoD components requiring NSS-level protection | Supported on DoW path with additional controls |

## IL2

IL2 is the baseline for non-sensitive federal data. Systems authorized at IL2 can handle information that is publicly releasable or internal-use-only but does not contain CUI. This maps closely to the FedRAMP Moderate baseline. Game Warden's FedRAMP path supports IL2 workloads.

## IL4

IL4 is the most common impact level for defense software vendors. It covers Controlled Unclassified Information (CUI) and Covered Defense Information (CDI)—data that is not classified but requires protection under law, regulation, or government policy. Most DoD program offices require IL4 for vendor software handling contract information, technical data, or operational data.

Game Warden's DoW path is fully optimized for IL4 deployments. Vendors at IL4 must complete a BoE and receive an ATO but can inherit a substantial portion of their security controls from the Game Warden system boundary.

## IL5

IL5 covers National Security System (NSS) data and higher-sensitivity CUI. It is required for systems that support DoD mission operations or handle data that, if disclosed, could damage national security. IL5 systems must meet all IL4 requirements plus additional controls around personnel vetting, physical security, encryption standards, and network isolation.

<Callout type="important">
  IL5 is the highest impact level Game Warden supports and requires additional compliance steps. If your program requires IL5, contact your Second Front account representative before beginning onboarding. Review [IL5 Compliance Requirements](/getting-started/dow-il5) for the full list of additional obligations.
</Callout>

## How Game Warden Supports Each Level

Game Warden manages the underlying infrastructure security controls for each impact level, allowing vendors to focus on their application-layer controls. Specifically, GW handles:

- Physical security of data centers
- Network isolation and segmentation (NIPRNET connectivity for IL4/IL5)
- Baseline OS and container hardening
- Continuous monitoring and SIEM integration
- FedRAMP/RMF control documentation for inherited controls

Vendors remain responsible for application-level security controls, including authentication, authorization, data encryption at the application layer, and secure coding practices.
