Game Warden's FedRAMP environment uses role-based access control (RBAC) to ensure that users have only the permissions necessary to perform their job functions.

## Role Types

| Role | Description |
|---|---|
| **Org Admin** | Full administrative access. Can manage members, create and delete applications, configure settings, and view all audit logs. |
| **Developer** | Can submit and manage applications, view findings, upload documents to App Central, and run pipeline scans. Cannot manage organization members or billing. |
| **Auditor** | Read-only access to all application data, findings, BoE documents, and audit logs. Intended for compliance officers and assessors. |
| **Read-Only** | Read-only access to application status and non-sensitive metadata. Cannot view security findings or BoE documents. |

## Assigning Roles

Only Org Admins can assign or modify roles.

<Steps>
  <Step n={1} title="Navigate to Organization Settings">
    In the Game Warden portal, click your organization name and select **Organization Settings**.
  </Step>
  <Step n={2} title="Open the Members Tab">
    Click the **Members** tab to see all current members and their roles.
  </Step>
  <Step n={3} title="Modify a Member's Role">
    Click the three-dot menu next to a member's name and select **Change Role**. Select the new role and click **Save**.
  </Step>
</Steps>

## MFA Requirements

Multi-factor authentication is mandatory for all users in the FedRAMP environment. Game Warden supports:

- **TOTP authenticator apps** (Google Authenticator, Authy, Microsoft Authenticator)
- **Hardware security keys** (FIDO2/WebAuthn compliant devices such as YubiKey)

SMS-based MFA is not permitted in the FedRAMP environment per NIST SP 800-63B requirements.

## Session Timeout Policies

| Session Type | Timeout |
|---|---|
| Inactive session | 15 minutes |
| Maximum session duration | 8 hours |
| Privileged action re-authentication | Required for destructive actions |

<Callout type="note">
  These session policies are enforced at the platform level and cannot be modified by Org Admins. They reflect FedRAMP Moderate requirements under control AC-12.
</Callout>

## Least Privilege Guidance

- Assign the **Developer** role to engineers who need to submit and manage applications. Do not grant Org Admin to all engineers.
- Use the **Auditor** role for compliance officers and third-party assessors instead of granting Developer access.
- Review member roles quarterly and remove access for team members who have left the project.
- Do not share login credentials. Each team member must have their own account.
