This page provides a technical overview of the Commercial deployment path, including architecture, supported regions, compliance posture, and known limitations compared to the DoW and FedRAMP paths.

## Architecture Overview

Commercial deployments run on Game Warden's managed Kubernetes infrastructure hosted in AWS commercial regions. Each organization receives a dedicated namespace within the shared cluster, with Kubernetes RBAC and network policies enforcing tenant isolation.

Key architectural components:
- **Kubernetes**: EKS-based clusters with automated upgrades and patching
- **Ingress**: NGINX ingress controller with TLS termination and WAF integration
- **Service mesh**: Istio for inter-service mTLS and traffic management
- **Registry**: Harbor private container registry with Trivy-based vulnerability scanning
- **Secrets**: Vault-based secrets management integrated with Kubernetes workloads
- **Observability**: Prometheus + Grafana for metrics, Loki for log aggregation

## Supported Cloud Regions

| Region | AWS Region Code | Status |
|---|---|---|
| US East (N. Virginia) | us-east-1 | Generally Available |
| US West (Oregon) | us-west-2 | Generally Available |
| US Gov East | us-gov-east-1 | FedRAMP/DoW path only |

Commercial deployments default to us-east-1. Contact your Second Front representative to request us-west-2 or multi-region deployments.

## Compliance Posture

Game Warden's commercial environment is audited annually against:

- **SOC2 Type II** — Security, Availability, and Confidentiality trust service criteria
- **ISO 27001** — Information security management system standard

SOC2 and ISO 27001 reports are available to customers under NDA.

## Limitations vs. DoW and FedRAMP

The commercial path does not include:
- NIPRNET network connectivity
- DoD ATO or Deployment Passport
- FedRAMP Moderate control inheritance
- CAC/PIV authentication integration (available as a custom configuration on request)
- IL4/IL5 data classification support

If any of these are required by your program, you must use the DoW or FedRAMP path instead.
