This page tracks updates to the Game Warden platform, App Central, and associated tooling. Releases are deployed on a rolling basis. Critical security patches may be deployed outside the regular release schedule.

## June 2026

### v3.14.0 — June 10, 2026

#### New Features

- **Managed Redis Cluster Mode**: Redis managed service now supports cluster mode for high-throughput workloads requiring horizontal scaling. Available for all environments. Request cluster mode via App Central under **Services > Redis > Configuration**.
- **SBOM Upload in App Central**: Vendors targeting IL4/IL5 can now upload Software Bill of Materials (SBOM) files directly in App Central as part of the BoE submission flow. Supported formats: CycloneDX JSON, SPDX 2.3.
- **Grafana Alerting Improvements**: Pre-built alert templates now include a "New Critical Finding Detected" alert that triggers immediately when a Critical CVE is identified by the pipeline scanner. Enable under **Observability > Alerts > Templates**.

#### Improvements

- Pipeline scan results now include a remediation suggestion for each CVE finding, sourced from the NVD advisory database. The suggestion is displayed in the finding detail view in App Central.
- BoE completion percentage now updates in real time as reviewers mark sections complete, without requiring a page refresh.
- Harbor image detail pages now display the Cosign signature verification status alongside the vulnerability scan summary.
- Vault secret rotation is now self-service for non-dynamic credentials. Org Admins can trigger manual rotation from **Organization Settings > Secrets**.

#### Bug Fixes

- Fixed an issue where pipeline runs would remain in "In Progress" status indefinitely if the Harbor webhook delivery failed. Failed webhooks now retry automatically with exponential backoff.
- Fixed a display issue in App Central where finding severity badges were rendered incorrectly in Safari on macOS.
- Resolved a race condition in the namespace provisioning workflow that occasionally caused NetworkPolicy creation to fail silently during org setup.

## May 2026

### v3.13.0 — May 20, 2026

<Callout type="important">
  **Breaking Change**: The `gw-standard` storage class now uses GP3 EBS volumes instead of GP2. Existing PVCs are not affected. New PVC provisioning requests using `gw-standard` will receive GP3 volumes, which have different IOPS baseline characteristics. Review your application's storage I/O requirements if you are provisioning new volumes.
</Callout>

#### New Features

- **Jaeger Distributed Tracing (GA)**: Distributed tracing via Jaeger is now generally available for all environments. Enable tracing for your namespace by submitting a support ticket requesting Jaeger integration. Trace data is retained for 7 days.
- **FedRAMP POA&M Dashboard**: A new dedicated POA&M dashboard is available for FedRAMP path tenants, showing all open accepted risks, their target remediation dates, and days remaining. Access under **Applications > [App] > BoE > POA&M**.
- **API Key Scoping**: Organization API keys can now be scoped to specific applications in addition to organization-wide access. Useful for restricting CI/CD pipeline keys to a single application's registry namespace.

#### Improvements

- Upgraded Kubernetes cluster version to 1.30 across all environments. Tenants should verify their Kubernetes API version compatibility. The Kubernetes 1.27 API deprecations apply; deprecated API versions will return warnings in pipeline logs.
- Semgrep SAST ruleset updated to include 47 new rules targeting misconfigured Kubernetes RBAC patterns and Helm chart security issues.
- App Central onboarding status page now shows estimated days remaining for each active onboarding phase based on historical data.
- Harbor vulnerability scan reports can now be exported as PDF from the image detail page.

#### Bug Fixes

- Fixed an issue where Org Admin role changes did not immediately reflect in the portal navigation without a full page reload.
- Resolved incorrect SLA countdown display for findings detected on weekends (SLA timer was counting calendar days instead of business days for Medium/Low severity).
- Fixed a bug where the BoE "Submit for Review" button was not visible when the browser zoom level was set above 125%.

### v3.12.0 — May 6, 2026

#### New Features

- **Managed PostgreSQL 15 GA**: PostgreSQL 15 is now generally available as a managed service option. New database instances default to PG15. Existing PG14 instances are not automatically upgraded; contact support to schedule an upgrade.
- **pgvector Extension Support**: Managed PostgreSQL instances now support the `pgvector` extension for storing and querying vector embeddings. Useful for AI/ML workloads using semantic search or RAG patterns. Request `pgvector` enablement via a support ticket.

#### Improvements

- Loki log retention for DoW IL5 environments extended to 3 years, from the previous 2-year retention period, to meet updated DISA audit log requirements.
- The GW CLI (`gw`) now includes a `gw pipeline status` command that outputs the current pipeline run status and scan results for a given image tag.

#### Bug Fixes

- Fixed an issue where deleting an application in App Central did not remove the associated Harbor project, causing orphaned registry storage.
- Resolved a Vault Agent injection failure that occurred when pod names contained uppercase letters.

## April 2026

### v3.11.0 — April 15, 2026

<Callout type="important">
  **Breaking Change**: Prisma Cloud (Twistlock) scanner version upgraded from 32.x to 33.x for all DoW environments. The updated scanner includes an expanded vulnerability database and may surface new findings that were not detected previously. Review your open findings after the upgrade and remediate or document any new Critical/High CVEs within SLA.
</Callout>

#### New Features

- **Self-Service Egress Allowlisting (Commercial/FedRAMP)**: Vendors on Commercial and FedRAMP paths can now request egress allowlist rules directly through App Central under **Applications > [App] > Network > Egress Rules**, without filing a support ticket. Rules are automatically reviewed and applied within 4 business hours.
- **IL5 BoE Template**: A new IL5-specific BoE template is now available in App Central for organizations onboarding to IL5. The template pre-populates IL5-specific control documentation requirements and includes guidance for CAC/PIV integration sections.

#### Improvements

- App Central now sends a weekly digest email summarizing open findings, POA&M items approaching their deadline, and BoE sections awaiting action. Opt in or out under **Profile > Notification Settings**.
- ArgoCD deployment history in App Central now shows the deployed image digest alongside the image tag, making it easier to correlate deployments with specific pipeline scan results.
- Trivy vulnerability database is now updated every 6 hours (previously every 24 hours), reducing the window between CVE disclosure and detection.

#### Bug Fixes

- Fixed an issue where cert-manager failed to renew certificates for custom domains when the DNS challenge provider returned a transient error. cert-manager now retries the challenge with backoff.
- Resolved an App Central performance issue where loading the Findings tab for applications with more than 500 historical findings caused the page to become unresponsive.
- Fixed incorrect RBAC assignment that allowed Developer-role users to view Vault audit logs. Vault audit log access is now restricted to Org Admin and Auditor roles.

### v3.10.2 — April 3, 2026

#### Bug Fixes

- Emergency patch: Upgraded Vault to 1.16.3 to address CVE-2026-0987 (CVSS 9.2) — authentication bypass in certain token renewal paths. All tenants are automatically protected. No action required.
- Fixed a regression introduced in v3.10.0 where pod startup times increased by 8–12 seconds due to an unintended change in Vault Agent sidecar injection ordering.
